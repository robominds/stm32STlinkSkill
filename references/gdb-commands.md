# GDB Command Reference for STM32 Embedded Debugging

Quick reference for `arm-none-eabi-gdb` when connected to an STM32 via OpenOCD.

---

## Session Setup (handled by debug.sh)

The `.gdbinit-stm32` file generated by `debug.sh` runs these automatically:

```gdb
target remote localhost:3333    # Connect to OpenOCD GDB server
monitor reset halt              # Reset and halt the target
break main                      # Breakpoint at main()
```

---

## Execution Control

| Command | Short | Description |
|---|---|---|
| `continue` | `c` | Resume execution |
| `step` | `s` | Step one source line (into functions) |
| `next` | `n` | Step one source line (over functions) |
| `stepi` | `si` | Step one machine instruction (into calls) |
| `nexti` | `ni` | Step one machine instruction (over calls) |
| `finish` | `fin` | Run until current function returns |
| `until <line>` | `u <line>` | Run until source line |
| `return` | | Return from current function immediately |

---

## Breakpoints and Watchpoints

| Command | Description |
|---|---|
| `break main` | Breakpoint at function `main` |
| `break file.c:42` | Breakpoint at file and line number |
| `break *0x08001234` | Hardware breakpoint at memory address |
| `hbreak main` | Explicitly hardware breakpoint (use for flash) |
| `info breakpoints` | List all breakpoints |
| `delete 1` | Delete breakpoint number 1 |
| `disable 1` | Disable breakpoint 1 without deleting |
| `enable 1` | Re-enable breakpoint 1 |
| `watch variable` | Data watchpoint: break when `variable` is written |
| `rwatch variable` | Break when `variable` is read |
| `awatch variable` | Break on read or write |
| `info watchpoints` | List all watchpoints |

**Important:** STM32 Cortex-M4 supports 6 hardware breakpoints and 4 watchpoints (varies by family). Software breakpoints (which temporarily patch flash) typically don't work on STM32 — use `hbreak` instead of `break` for code in flash.

## Inspecting Variables and Memory

| Command | Description |
|---|---|
| `print variable` | Print variable in default format |
| `print/x variable` | Print as hexadecimal |
| `print/d variable` | Print as signed decimal |
| `print/u variable` | Print as unsigned decimal |
| `print/b variable` | Print as binary |
| `print *pointer` | Dereference and print pointer |
| `print array[0]@10` | Print 10 elements starting at index 0 |
| `display variable` | Auto-print variable at every stop |
| `undisplay 1` | Remove auto-display number 1 |
| `info display` | List all auto-displays |

### Memory Examination (x command)

```gdb
x/10xw 0x20000000     # 10 words (32-bit) as hex at RAM start
x/4xb &variable       # 4 bytes at variable's address
x/1xh 0x40020014      # 1 halfword (16-bit) as hex
x/10i $pc             # Disassemble 10 instructions at PC
x/s 0x08002000        # Read as null-terminated string
```

Format: `x/<count><format><size> <address>`
- Formats: `x` hex, `d` decimal, `u` unsigned, `b` binary, `i` instruction, `s` string
- Sizes: `b` byte (8), `h` halfword (16), `w` word (32), `g` giant (64)

## CPU Registers

| Command | Description |
|---|---|
| `info registers` | Show all CPU registers |
| `info registers r0` | Show specific register |
| `print $pc` | Print Program Counter |
| `print/x $sp` | Print Stack Pointer as hex |
| `print/x $lr` | Print Link Register as hex |
| `print $xpsr` | Print combined Program Status Register |
| `set $r0 = 42` | Write to register |

ARM Cortex-M registers: `r0`–`r12`, `sp` (r13), `lr` (r14), `pc` (r15), `xpsr`

|---|---|---|
| `backtrace` | `bt` | Show full call stack |
| `backtrace 5` | `bt 5` | Show top 5 frames only |
| `frame 2` | `f 2` | Select stack frame 2 |
| `info frame` | | Details of current frame |
| `info locals` | | Local variables in current frame |
| `info args` | | Function arguments in current frame |
| `up` | | Move up one stack frame |
| `down` | | Move down one stack frame |

## OpenOCD Monitor Commands

Send TCL commands directly to OpenOCD from within GDB:

| Command | Description |
|---|---|
| `monitor reset halt` | Reset target and halt at reset vector |
| `monitor reset run` | Reset target and let it run freely |
| `monitor halt` | Halt target immediately |
| `monitor resume` | Resume target execution |
| `monitor poll` | Show current target state |
| `monitor reg` | Show all registers via OpenOCD |
| `monitor flash list` | List flash memory regions |
| `monitor flash info 0` | Show info for flash bank 0 |
| `monitor mdw 0x20000000` | Read one word from address |
| `monitor mdw 0x20000000 8` | Read 8 words from address |
| `monitor mww 0x20000000 0xDEADBEEF` | Write word to address |
| `monitor targets` | Show target list and state |

## Flashing from Within GDB

```gdb
# Load firmware into flash from an active debug session
load

# Verify flash contents match the ELF
compare-sections

# Reset after loading
monitor reset halt
```

## Peripheral Register Access

STM32 peripherals are memory-mapped. Access them directly in GDB:

```gdb
# Read GPIOA ODR (Output Data Register) — STM32F4 example
print/x *(unsigned int*)0x40020014

# Set GPIOA BSRR bit 5 (set PA5 high)
set *(unsigned int*)0x40020018 = 0x00000020

# Clear GPIOA BSRR bit 5 (set PA5 low) via BSRR reset bits [31:16]
set *(unsigned int*)0x40020018 = 0x00200000

# Read USART1 ISR (interrupt and status register)
print/x *(unsigned int*)0x40011000

# Read RCC AHB1ENR (clock enable register)
print/x *(unsigned int*)0x40023830
```

**Common STM32F4 peripheral base addresses:**

| Peripheral | Base Address |
|---|---|
| GPIOA | 0x40020000 |
| GPIOB | 0x40020400 |
| GPIOC | 0x40020800 |
| USART1 | 0x40011000 |
| USART2 | 0x40004400 |
| SPI1 | 0x40013000 |
| I2C1 | 0x40005400 |
| TIM1 | 0x40010000 |
| TIM2 | 0x40000000 |
| RCC | 0x40023800 |
| FLASH | 0x40023C00 |
| SysTick | 0xE000E010 |
| NVIC | 0xE000E100 |
| SCB | 0xE000ED00 |
| DWT | 0xE0001000 |
| ITM | 0xE0000000 |

Consult your device's Reference Manual for the exact peripheral map.

## HardFault Analysis

When the target crashes into a HardFault handler, use these commands to diagnose:

```gdb
# 1. Check the call stack
backtrace

# 2. Read Configurable Fault Status Register
print/x *(unsigned int*)0xE000ED28    # CFSR
# CFSR bits:
#   [7:0]   MMFSR — MemManage Fault
#   [15:8]  BFSR  — BusFault
#   [31:16] UFSR  — UsageFault (common: INVSTATE=invalid Thumb state, UNALIGNED)

# 3. Hard Fault Status Register
print/x *(unsigned int*)0xE000ED2C    # HFSR
# Bit 30 (FORCED) = escalated configurable fault
# Bit 31 (DEBUGEVT) = debug event caused fault

# 4. Fault addresses (only valid when corresponding VALID bit is set in CFSR)
print/x *(unsigned int*)0xE000ED34    # MMFAR — MemManage fault address
print/x *(unsigned int*)0xE000ED38    # BFAR  — BusFault address
```

**Common fault causes:**
- `IACCVIOL` (MMFSR bit 0) — instruction access from non-executable region
- `DACCVIOL` (MMFSR bit 1) — data access violation
- `STKERR` (BFSR bit 4) — stack overflow during push
- `INVSTATE` (UFSR bit 1) — invalid ARM state (often a bad function pointer or null call)
- `UNALIGNED` (UFSR bit 8) — unaligned memory access

## Semihosting (printf over debug)

If your firmware uses semihosting for standard output:

```gdb
monitor arm semihosting enable
```

After enabling, `printf()` output from the firmware appears in the OpenOCD terminal.

---

## Useful GDB Settings

```gdb
set print pretty on           # Structured output for structs/unions
set print array on            # Array-friendly printing
set print array-indexes on    # Show array indices
set pagination off            # No "Press Enter" prompts
set confirm off               # No "Are you sure?" prompts
set mem inaccessible-by-default off  # Don't error on unmapped memory regions
```

---

## Exiting

```gdb
quit                    # Exit GDB (OpenOCD continues in background)
monitor shutdown        # Also stop OpenOCD
```

When using `debug.sh`, OpenOCD is automatically stopped when GDB exits.

---

## GDB Init Script Tips

For repeated tasks, create a `.gdbinit` in your project:

```gdb
# Custom .gdbinit for STM32F4 project
set print pretty on
set pagination off

target remote localhost:3333
monitor reset halt

# Project-specific breakpoints
break HardFault_Handler
break app_main

# Auto-display key variables
display/x uwTick
display motorSpeed
```

Run with:
```bash
arm-none-eabi-gdb -x .gdbinit build/firmware.elf
```
